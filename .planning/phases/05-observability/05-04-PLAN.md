<title>Create correlation ID middleware for API service</title>
<read_first>
- services/api/internal/middleware/auth.go
- services/api/internal/middleware/security.go
- pkg/logger/correlation.go
</read_first>
<action>
Create new file `services/api/internal/middleware/correlation.go` with package `middleware`.

Define function:

```go
func CorrelationID() func(http.Handler) http.Handler
```

The middleware must:
1. Check incoming request for `X-Correlation-ID` header — use if non-empty.
2. Otherwise generate a new UUID via `uuid.New().String()`.
3. Store the correlation ID in context via `logger.WithCorrelationID(r.Context(), id)`.
4. Set `X-Correlation-ID` response header with the ID (so clients can reference it).
5. Call `next.ServeHTTP(w, r.WithContext(ctx))`.

Imports: `net/http`, `github.com/google/uuid`, `github.com/f1xgun/onevoice/pkg/logger`.

Note: `github.com/google/uuid` is already a dependency in `services/api/go.mod`.
</action>

<action>
In `services/orchestrator/cmd/main.go`, the orchestrator also receives `X-Correlation-ID` from the API proxy. While task 5-04-04 handles extracting it in the chat handler, add a simple middleware to the orchestrator's chi router that stores it in context for non-chat routes and logging.

After `r.Use(chimiddleware.RequestID)` (line 73), add a small inline middleware:

```go
r.Use(func(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		ctx := r.Context()
		if corrID := r.Header.Get("X-Correlation-ID"); corrID != "" {
			ctx = logger.WithCorrelationID(ctx, corrID)
		}
		next.ServeHTTP(w, r.WithContext(ctx))
	})
})
```

Add `"github.com/f1xgun/onevoice/pkg/logger"` to the imports. Also add `"net/http"` if not already imported.

Note: `net/http` is already imported.
</action>
