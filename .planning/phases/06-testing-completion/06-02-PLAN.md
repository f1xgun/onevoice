<action>
Append to `pkg/health/health_test.go`:

1. **TestReadyHandler_AllFailing** — register 2 checks that both return errors. Verify:
   - Status 503
   - `"status": "not_ready"`
   - Both check names appear in `checks` map with their error messages

2. **TestReadyHandler_ContextTimeout** — register a check that blocks on `<-ctx.Done()` then returns `ctx.Err()`. Verify:
   - Status 503
   - `"status": "not_ready"`
   - The timed-out check reports a context error message

3. **TestLiveHandler_ContentType** — verify response `Content-Type` is `application/json`.

4. **TestReadyHandler_ContentType** — verify response `Content-Type` is `application/json`.

5. **TestAddCheck_ConcurrentSafety** — spawn 10 goroutines that each call `AddCheck` with unique names concurrently. Then call `ReadyHandler` and verify all 10 checks appear in the response. This validates the `sync.RWMutex` protection.

6. **TestReadyHandler_MixedResults** — register 3 checks: 2 healthy + 1 failing. Verify:
   - Status 503
   - The 2 healthy checks show `"ok"`
   - The 1 failing check shows the error message
   - `"status": "not_ready"`

For the timeout test, use a short context timeout via `httptest.NewRequest` with a pre-canceled or short-timeout context, or rely on the 5-second internal timeout by using a check that sleeps longer than 5 seconds (use `time.After` + `ctx.Done()` select pattern to avoid actually sleeping).
</action>
