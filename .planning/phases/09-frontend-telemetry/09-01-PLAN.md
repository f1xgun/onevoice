    1. Create `services/frontend/lib/telemetry.ts`:
       - Define `TelemetryEvent` interface: `{ eventType: string; page: string; action: string; correlationId?: string; metadata?: Record<string, string>; timestamp: string }`
       - Module-level `let buffer: TelemetryEvent[] = []`
       - Module-level `let flushTimer: ReturnType<typeof setTimeout> | null = null`
       - `const BATCH_INTERVAL = 5000` (5 seconds)
       - `const MAX_BATCH_SIZE = 50`
       - `function trackEvent(eventType: string, action: string, opts?: { page?: string; correlationId?: string; metadata?: Record<string, string> })`:
         - Push event to buffer with `timestamp: new Date().toISOString()`, `page: opts?.page ?? (typeof window !== 'undefined' ? window.location.pathname : '')`
         - If buffer.length >= MAX_BATCH_SIZE, call `flushTelemetry()` immediately
         - Otherwise, if no flushTimer is running, set one for BATCH_INTERVAL
       - `async function flushTelemetry()`:
         - If buffer is empty, return
         - Take current buffer, reset to empty array
         - Clear flushTimer
         - POST to `/api/v1/telemetry` using the `api` axios instance (already has auth token)
         - Fire-and-forget: catch errors silently (telemetry must never break the app)
         - Use `navigator.sendBeacon` as fallback in `visibilitychange` event for page unload
       - Register `visibilitychange` listener: if `document.visibilityState === 'hidden'`, flush via sendBeacon with JSON blob to `/api/v1/telemetry`
       - Export: `trackEvent`, `flushTelemetry`

    2. Update `services/frontend/lib/api.ts`:
       - Import `{ trackEvent }` from `./telemetry`
       - In the response error interceptor, BEFORE the 401 check, capture correlation_id:
         ```typescript
         const correlationId = error.response?.headers?.['x-correlation-id'];
         if (error.response && correlationId) {
           trackEvent('api_error', `${error.response.status} ${original?.method?.toUpperCase()} ${url}`, {
             correlationId,
             metadata: { status: String(error.response.status), url },
           });
         }
         ```
       - This goes right after `const url = original?.url ?? '';` and before the `isAuthEndpoint` check
       - Telemetry calls themselves should not trigger telemetry (avoid infinite loop): check `if (url.includes('/telemetry'))` and skip tracking for telemetry endpoint errors
  </action>
