# PLAN-2.4: Replace panic() in production handlers with error returns

## Summary

Replace all 10 `panic()` calls across 8 constructors in the API service with `error` return values. Update `services/api/cmd/main.go` to handle constructor errors and propagate them through the existing `run() error` flow. This ensures malformed startup configuration results in a clean log + exit(1) instead of a stack trace crash.
