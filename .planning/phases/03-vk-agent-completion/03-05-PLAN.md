## Overview

Create integration tests for all VK client methods using an `httptest.Server` that mimics VK API JSON responses. Tests cover all 9 VK tools (3 existing + 6 new) across success, permanent error (code 5), rate-limit error (code 6), and network failure scenarios. The VK SDK's internal HTTP client is redirected to the test server.

<read_first>

- services/agent-vk/go.mod (vksdk version: v3.2.2)

<action>
1. In `services/agent-vk/internal/vk/client.go`, check if the vksdk `vkapi.VK` struct has a `MethodURL` field or equivalent that allows overriding the API base URL. The vksdk v3 uses `vk.MethodURL` (string field) defaulting to `"https://api.vk.com/method/"`.

2. Add an `Option` pattern or a separate constructor for tests:
   ```go
   // NewWithBaseURL creates a Client pointing at a custom API base URL (for testing).
   func NewWithBaseURL(accessToken, baseURL string) *Client {
       vk := vkapi.NewVK(accessToken)
       vk.MethodURL = baseURL
       return &Client{vk: vk}
   }
   ```

   If `MethodURL` is not directly accessible, use `vk.Client` field to inject a custom `http.Client` with a transport that rewrites URLs:
   ```go
   func NewWithBaseURL(accessToken, baseURL string) *Client {
       vk := vkapi.NewVK(accessToken)
       vk.Client.Transport = &rewriteTransport{base: baseURL}
       return &Client{vk: vk}
   }

   type rewriteTransport struct {
       base string
   }

   func (t *rewriteTransport) RoundTrip(req *http.Request) (*http.Response, error) {
       req.URL.Scheme = ""
       req.URL.Host = ""
       newURL := t.base + req.URL.Path + "?" + req.URL.RawQuery
       newReq, _ := http.NewRequestWithContext(req.Context(), req.Method, newURL, req.Body)
       newReq.Header = req.Header
       return http.DefaultTransport.RoundTrip(newReq)
   }
   ```

   **Implementation note:** Try `vk.MethodURL` first — it's cleaner. Only fall back to transport rewriting if MethodURL doesn't work.
</action>
