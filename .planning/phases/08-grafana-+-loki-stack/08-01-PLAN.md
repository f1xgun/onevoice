Create the observability/ directory tree with these config files:

1. **observability/loki/loki-config.yml** — Loki v2 config:
   - auth_enabled: false
   - server http_listen_port: 3100
   - ingester: lifecycler ring kvstore=inmemory, replication_factor=1
   - schema_config: configs with store=tsdb, object_store=filesystem, schema=v13, index prefix=loki_index_ period=24h
   - storage_config: tsdb_shipper active_index_directory=/loki/index, cache_location=/loki/cache; filesystem directory=/loki/chunks
   - compactor: working_directory=/loki/compactor
   - limits_config: reject_old_samples=true, reject_old_samples_max_age=168h

2. **observability/promtail/promtail-config.yml** — Promtail config:
   - server http_listen_port: 9080
   - positions filename: /tmp/positions.yaml
   - clients url: http://loki:3100/loki/api/v1/push
   - scrape_configs job_name=docker:
     - static_configs targets=localhost, labels job=docker, __path__=/var/log/containers/*.log
     - pipeline_stages: docker {} stage to parse Docker JSON log format
     - relabel_configs: use __meta_docker_container_name to set container label

   Actually use Docker service discovery:
   - docker_sd_configs host: unix:///var/run/docker.sock, refresh_interval: 5s
   - relabel_configs: source_labels __meta_docker_container_name, regex /(.*), target_label container
   - pipeline_stages: docker {} (parses Docker JSON log lines)

3. **observability/prometheus/prometheus.yml** — Prometheus config:
   - global scrape_interval: 15s, evaluation_interval: 15s
   - scrape_configs:
     - job_name: api, static_configs targets: ["api:8080"], metrics_path: /metrics
     - job_name: orchestrator, static_configs targets: ["orchestrator:8090"], metrics_path: /metrics

4. **observability/grafana/provisioning/datasources/datasources.yml** — Grafana datasource provisioning:
   - apiVersion: 1
   - datasources:
     - name: Loki, type: loki, access: proxy, url: http://loki:3100, isDefault: true
     - name: Prometheus, type: prometheus, access: proxy, url: http://prometheus:9090

5. **observability/grafana/provisioning/dashboards/dashboards.yml** — Grafana dashboard provisioning:
   - apiVersion: 1
   - providers:
     - name: default, orgId: 1, folder: "", type: file, disableDeletion: false, editable: true
     - options path: /var/lib/grafana/dashboards

Create docker-compose.observability.yml as a compose overlay that extends docker-compose.yml.

Services to define:

1. **loki** — grafana/loki:3.0.0
   - container_name: onevoice-loki
   - command: -config.file=/etc/loki/loki-config.yml
   - volumes: ./observability/loki/loki-config.yml:/etc/loki/loki-config.yml:ro, loki_data:/loki
   - ports: 3100:3100
   - healthcheck: wget --no-verbose --tries=1 --spider http://localhost:3100/ready || exit 1
   - networks: onevoice-network
   - restart: unless-stopped

2. **promtail** — grafana/promtail:3.0.0
   - container_name: onevoice-promtail
   - command: -config.file=/etc/promtail/promtail-config.yml
   - volumes:
     - ./observability/promtail/promtail-config.yml:/etc/promtail/promtail-config.yml:ro
     - /var/run/docker.sock:/var/run/docker.sock:ro (for Docker service discovery)
   - depends_on: loki (condition: service_healthy)
   - networks: onevoice-network
   - restart: unless-stopped

3. **prometheus** — prom/prometheus:v2.52.0
   - container_name: onevoice-prometheus
   - command: --config.file=/etc/prometheus/prometheus.yml --storage.tsdb.retention.time=7d
   - volumes: ./observability/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml:ro, prometheus_data:/prometheus
   - ports: 9090:9090
   - healthcheck: wget --no-verbose --tries=1 --spider http://localhost:9090/-/healthy || exit 1
   - networks: onevoice-network
   - restart: unless-stopped

4. **grafana** — grafana/grafana:11.0.0
   - container_name: onevoice-grafana
   - environment: GF_SECURITY_ADMIN_USER=admin, GF_SECURITY_ADMIN_PASSWORD=admin, GF_USERS_ALLOW_SIGN_UP=false
   - volumes:
     - ./observability/grafana/provisioning:/etc/grafana/provisioning:ro
     - ./observability/grafana/dashboards:/var/lib/grafana/dashboards:ro
     - grafana_data:/var/lib/grafana
   - ports: 3001:3000 (Grafana on 3001 to avoid conflict with frontend's 3000)
   - depends_on: loki (service_healthy), prometheus (service_healthy)
   - healthcheck: wget --no-verbose --tries=1 --spider http://localhost:3000/api/health || exit 1
   - networks: onevoice-network
   - restart: unless-stopped

Volumes section: loki_data, prometheus_data, grafana_data (all local driver)

Networks section: reference onevoice-network as external: false (or just list it; compose overlay merges with main).

IMPORTANT: Do NOT redefine services from docker-compose.yml. The overlay only adds new services. Use the same network name "onevoice-network" so all services can communicate.
