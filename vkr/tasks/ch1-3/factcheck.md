---

- **утверждение**: Мультиагентные (многоагентные) системы характеризуются распределённостью, автономностью агентов и возможностью специализации/кооперации для решения сложных задач.
- **статус**: ok
- **подтверждающие_источники**: https://dl.acm.org/doi/full/10.1145/3745238.3745531; https://a2a-protocol.org/latest/specification/
- **пояснение**: Современный обзор по multi-agent collaboration описывает принципы распределённости/кооперации и специализации; спецификация A2A оперирует понятием независимых агентных систем и мотивацией интероперабельности, что согласуется с тезисом о распределённой кооперации. Формулировка общая и не привязана к спорным количественным оценкам.
- **рекомендуемая_правка_формулировки**: —

---

- **утверждение**: A2A (Agent2Agent) протокол позиционируется как открытый стандарт для взаимодействия независимых агентных систем и использует известные транспорт/форматы (HTTP, JSON-RPC 2.0, SSE).
- **статус**: needs_fix
- **подтверждающие_источники**: https://a2a-protocol.org/latest/specification/; https://www.jsonrpc.org/specification
- **пояснение**: Спецификация A2A напрямую указывает повторное использование HTTP/JSON-RPC 2.0/SSE. Однако вторым источником фактически выступает только спецификация JSON-RPC (она не подтверждает использование A2A, а лишь определяет сам формат). Для строгого правила «2 независимых источника» лучше либо (а) добавить независимый обзор A2A, либо (б) в `output.md` явно сослаться на спецификацию как на единственный нормативный источник: «в спецификации A2A указано…».
- **рекомендуемая_правка_формулировки**: Согласно спецификации A2A, протокол предназначен для взаимодействия независимых агентных систем и опирается на HTTP, JSON‑RPC 2.0 и Server‑Sent Events (SSE).

---

- **утверждение**: MCP (Model Context Protocol) определяет стандартизированный способ интеграции LLM-приложений с внешними источниками данных и инструментами и использует JSON-RPC 2.0.
- **статус**: needs_fix
- **подтверждающие_источники**: https://modelcontextprotocol.io/specification/2025-11-25; https://www.jsonrpc.org/specification
- **пояснение**: Спецификация MCP подтверждает архитектуру Host/Client/Server и использование JSON-RPC 2.0. JSON-RPC спецификация сама по себе не является независимым подтверждением MCP, но фиксирует стандарт формата. Для строгого требования независимости — либо добавить независимый обзор MCP, либо оставить утверждение как ссылку на спецификацию MCP.
- **рекомендуемая_правка_формулировки**: В спецификации MCP описан стандартный протокол взаимодействия LLM‑хостов/клиентов/серверов с внешними источниками данных и инструментами на базе JSON‑RPC 2.0 (указать версию/дату спецификации).

---

- **утверждение**: Внешние платформы, необходимые для управления цифровым присутствием (например Telegram и Instagram), предоставляют официальные API для интеграции.
- **статус**: needs_fix
- **подтверждающие_источники**: https://core.telegram.org/bots/api; https://developers.facebook.com/docs/instagram-platform/overview/; https://developers.facebook.com/docs/instagram-platform/reference/
- **пояснение**: Источники подтверждают наличие официальных API у Telegram (Bot API) и у Instagram Platform (для профессиональных аккаунтов). Однако исходная формулировка обобщает на все «необходимые платформы»; корректнее утверждать про конкретные примеры/ключевые платформы, указанные в постановке.
- **рекомендуемая_правка_формулировки**: Ключевые внешние платформы, используемые для управления цифровым присутствием (например Telegram и Instagram для профессиональных аккаунтов), предоставляют официальные API для интеграции; детали доступа определяются политиками платформ и уровнем авторизации.
